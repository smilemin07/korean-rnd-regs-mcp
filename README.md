# korean-rnd-regs-mcp

[![PyPI version](https://img.shields.io/pypi/v/korean-rnd-regs-mcp.svg)](https://pypi.org/project/korean-rnd-regs-mcp/)
[![Python](https://img.shields.io/pypi/pyversions/korean-rnd-regs-mcp.svg)](https://pypi.org/project/korean-rnd-regs-mcp/)
[![License: Apache 2.0](https://img.shields.io/badge/License-Apache_2.0-blue.svg)](LICENSE)

연구행정 관련 규정 검토용 MCP server. AI를 활용하여 아래와 같은 정보를 얻을 수 있습니다.
- 관련 규정 확인: 특정 사례의 규정 검토를 위해 확인이 필요한 규정의 조문 인출 
- 규정 검토: 특정 사례에 대한 다층적[법률 > 시행령 > 시행규칙 > 행정규칙(고시/훈령)] 규정 검토 결과
  - 동일한 프롬프트에 대해서도 사용하는 AI 모델에 따라 결과가 달라질 수 있습니다.

**대상 사용자**: 연구자, R&D전문기관 직원, 정부부처 R&D 담당 공무원

## 사용 예시

### 예시1: 관련 규정 확인

규정 검토가 필요한 사례를 입력하면, 검토가 필요한 세부 규정들을 출력해줍니다.

예시 prompt:

```
연구책임자 변경을 진행해야 하는 상황이야. 참고해야 할 세부 규정이 뭔지 알려줘.
```

---
  연구책임자 변경 관련 규정

  1. 협약 변경 경로 (일반적 변경)

  국가연구개발혁신법 제11조제2항 — 연구책임자 변경은 "연구개발과제 수행에 관한 중요한 사항"에 해당하여, 협약 당사자 간 협의를 거쳐 협약 내용을 변경해야 합니다.

  국가연구개발혁신법 시행령 제14조 — 구체적 절차를 규정합니다.
  - 제1항: 변경 사유와 내용을 상대방에게 사전에 문서로 명확히 알리고 상호 협의
  - 제2항 제2호: 경미한 사항(통보만으로 변경 가능)은 "연구책임자는 제외"라고 명시 → 즉, 연구책임자 변경은 경미한 사항이 아니므로 반드시 협의 절차 필요

  2. 특별평가 경로 (사유 발생 시)

  국가연구개발혁신법 제15조 — 다음 사유 발생 시 특별평가를 거쳐 연구책임자를 변경할 수 있습니다:
  - 부정행위 발생 (제1항 제1호)
  - 연구책임자 참여제한 확정 (제1항 제2호)
  - 연구개발 환경 변경 (제1항 제3호)
  - 연구기관/연구책임자 스스로 변경 요청 (제2항) — 연구개발 환경 변경 또는 수행 불가능 판단 시
  - 의무 불이행 또는 수행 불가능 인정 (제1항 제5호)

  국가연구개발혁신법 시행령 제31조 — 특별평가 절차:
  - 중앙행정기관 → 연구기관·연구책임자에게 평가 실시 시기, 사유, 소명자료 제출 시한 통보
  - 연구기관/연구책임자가 요청하는 경우: 시행규칙 별지 제9호서식(변경·중단 요청서) 제출

  3. 후속 조치 (연구개발비)

  연구개발비 사용 기준 제95조제1항 — 연구책임자 변경 시 학생인건비통합관리계정의 이관 처리 규정:
  - 같은 기관 내 다른 연구자로 변경 → 변경 후 연구책임자계정으로 이체
  - 다른 기관으로 변경(소속 변경) → 변경 후 연구개발기관의 과제계정으로 이체

  ---
  요약: 연구책임자 변경은 경미한 사항이 아니므로 시행령 제14조에 따른 사전 문서 통보 + 협약 당사자 간 협의가 필수이며, 부정행위·참여제한·수행불가 등 사유가 있으면 법 제15조의 특별평가 경로를 거칩니다. 변경 확정 후에는 연구개발비(특히 학생인건비) 이관 처리도 확인해야 합니다.
```

### 예시2: 규정 검토

규정 검토가 필요한 상황을 입력하면 다층적 규정 검토 결과가 출력됩니다.
- **'korean-rnd-regs-mcp'**, **'규정 검토'** 라는 문구가 prompt에 포함돼야 다층적 규정 검토가 진행됩니다.

예시 prompt:

```
나는 R&D전문기관에서 연구개발사업을 담당하는 직원이야. korean-rnd-regs-mcp를 활용해서 아래 사례에 대한 규정 검토를 진행해줘.
- 내가 담당하는 사업의 연구과제에서 간접비를 증액하고 싶다는 의견을 알려왔어.
  - 내가 확인해야 할 관련 규정을 보여줘.
  - 만일 R&D전문기관의 '승인'이 필요한 협약변경 건이라면, 내가 사업 담당자로서 확인해야 할 것들을 목록으로 제시해줘.
```

  - [ ] 보완 필요 여부: 제출 내용이 미흡하면 보완 요구 (제74조제2항)
  - [ ] 15일 이내 처리: 접수 후 15일 이내 승인/불승인 통보, 어려우면 기간 연장 통보 (제74조제3항)
  - [ ] 통합정보시스템 반영: 협약 변경 확정 시 NTIS 등 통합정보시스템에 변경사항 등록

## 설치 방법

### 사전 준비: LAW_API_KEY 발급

본 MCP 서버는 [국가법령정보 OpenAPI](https://open.law.go.kr)를 사용하여 연구행정 관련 규정을 불러옵니다.

#### 처음 발급받는 경우 (무료 / 5분 정도 소요)

1. https://open.law.go.kr 접속 → 회원가입
2. 로그인 → [마이페이지] → [API 신청] → "법령" 카테고리 신청 → '승인' 대기
3. 발급된 **OC 인증값**을 LAW_API_KEY로 설정

#### 이미 키를 보유하고 있는 경우 → 다음 단계 진행

---

### 설치

#### Option 1: Claude.ai(웹에서 Claude 사용)

1. claude.ai에 로그인
2. 왼쪽 사이드바에서 'Customize' 클릭
3. 'Connectors' 클릭
4. 돋보기 옆에 있는 '+' 클릭
5. 'Add custom connector' 클릭
6. Name: korean-rnd-regs-mcp
7. url: 아래 주소를 붙여넣기(mcp?oc=XX → XX 대신 법령정보센터에서 받은 API key 입력)
```
https://mcp.rndmanagers.org/mcp?oc=이_부분을_본인_법령정보센터_API키로_교체
```
8. 'Add' 클릭
9. 생성된 'korean-rnd-regs-mcp' 클릭
10. 우측에 'tool permissions'의 값을 'Needs approval'에서 'Always allow'로 수정
11. 완료

##### 설치 확인
새 채팅에서 아래 프롬프트를 입력했을 때, 'status=ok'가 출력되면 정상 설치
```
korean-rnd-regs-mcp 서버의 health 테스트를 진행해줘.
```

#### Option 2: Claude Code(터미널에서 Claude 사용)

1. uv가 설치돼 있어야 합니다.
  - uv 설치 여부를 터미널에서 확인 → 아래 명령어를 입력했을 때, uv의 버전이 출력돼야 함.
  ```
  uv --version
  ```
  - uv가 설치되지 않은 경우 → 아래 명령어를 입력해서 uv 설치를 진행
    - Windows
    ```
    powershell -c "irm https://astral.sh/uv/install.ps1 | iex"
    ```
    - Mac
    ```
    brew install uv
    ```
2. Claude Code 시작 → 아래 명령어 입력
```
/plugin marketplace add smilemin07/korean-rnd-regs-mcp
```
3. 아래 명령어 입력
```
/plugin install korean-rnd-regs@korean-rnd-regs-marketplace
```
4. 추가 선택 화면에서 mcp 활용 목적에 맞게 선택
  - 현재의 working directory에서만 사용하고 싶은 경우 → Local
  - 시스템 전체에서 사용하고 싶은 경우 → Global
5. 아래 명령어 입력
```
/reload-plugins
```
6. 완료

Claude Code에서 아래 프롬프트를 입력했을 때, 'status=ok'가 출력되면 정상 설치
```
korean-rnd-regs-mcp 서버의 health 테스트를 진행해줘.

## Troubleshooting

### Q1. AI가 "관련 조문을 찾을 수 없습니다"라고 답합니다

1. AI에게 도구 응답의 `errors` field 확인 요청 — API 인증 실패 등은 errors로 전달됨
2. `health` 도구로 `api_key_configured: true` 확인
3. LAW_API_KEY가 정확한지 [국가법령정보 OpenAPI 마이페이지](https://open.law.go.kr/LSO/mypage.do)에서 확인
4. 키 문자열 앞뒤 공백 제거

### Q2. AI가 조문 본문에 없는 부제나 요약을 만들어냅니다

- prompt에 "**편집하지 말고 그대로**" 또는 "**verbatim**" 이라는 문구를 추가

### Q3. 특정 조문이 검색되지 않습니다

- 검색어를 바꿔서 다시 시도 (띄어쓰기·약칭). 예: "연구개발비 사용 기준" → "연구개발비사용기준"
- 현재 버전에서는 '별표'에 있는 조문의 검색은 지원되지 않습니다.

### Q4. "auth_failed" 오류

- LAW_API_KEY가 비어있거나 잘못됨. open.law.go.kr 마이페이지 → API 키 재발급

### Q5. "rate_limited" 오류

- 일일 호출 제한 도달. 24시간 대기 후 재시도. 캐시(24h)가 동일 query 반복 호출을 줄여줍니다

## 보안

LAW_API_KEY는 국가법령정보 OpenAPI에서 무료로 발급받는 공개 법령 데이터 조회용 인증값입니다. 금융 API 키 등과는 성격이 다르지만, 다른 사람이 무단으로 사용할 수 없도록 관리하시기 바랍니다.
- 노출 시, 타인이 동일 키로 요청을 반복하여 사용에 영향을 줄 수 있습니다

### 서버의 보호 범위

- 도구 응답, 오류 메시지 등에 API 키가 표시되지 않도록 서버 내부에서 자동으로 숨김 처리합니다
- 법령정보 API와의 통신은 HTTPS 암호화를 사용합니다

### 사용자 주의사항

- API 키를 채팅, 스크린샷, SNS 등에 노출하지 마십시오
- 원격 접속(HTTP 모드) 사용 시 URL에 API 키가 포함됩니다 — 브라우저 기록, 터미널 기록에 남을 수 있으므로 공용 PC에서는 사용하지 마시기 바랍니다.
- `.env` 파일이 git commit에 포함되지 않도록 주의하십시오
- 타인이 운영하는 원격 서버를 이용하는 경우, 해당 서버 관리자는 기술적으로 API 키에 접근할 수 있습니다

### 키 노출 시 대처

키가 노출된 경우 [open.law.go.kr](https://open.law.go.kr)에서 기존 키를 삭제하고 새로 발급받을 수 있습니다.
